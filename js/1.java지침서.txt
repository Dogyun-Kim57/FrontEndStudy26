[ 변수와 상수, 데이터를 담는 그릇 만들기 ]
----------------------------------------------------------------------------------
변수는 데이터를 저장하는 그릇이라고 생각하면 됩니다.
마치 집에 있는 서랍장처럼, 각각의 서랍에 이름표를 붙여두고 필요한 물건을 넣어두는 것과 같아요.

// 변수 선언과 값 할당
let userName = "홍길동";
let userAge = 25;
let isStudent = true;

위 코드에서 userName, userAge, isStudent가 바로 변수의 이름(변수명)이고,
등호(=) 오른쪽에 있는 값들이 그 변수에 저장된 데이터입니다.

[ 변수 선언의 3가지 방법 ]
JavaScript에서 변수를 만드는 방법은 3가지가 있어요.

let - 값을 바꿀 수 있는 변수 (가장 많이 사용)
const - 값을 바꿀 수 없는 상수
var - 예전 방식 (요즘엔 거의 사용하지 않음)
----------------------------------------------------------------------------------
let score = 100;        // 점수는 바뀔 수 있으니까 let
const PI = 3.14159;     // 원주율은 절대 바뀌지 않으니까 const
score = 95;             // OK! let으로 선언했으니 변경 가능
// PI = 3.14;           // ERROR! const는 변경 불가능
----------------------------------------------------------------------------------
[ 변수명 짓기 규칙 ]
변수명은 아무거나 지을 수 없어요. 몇 가지 규칙이 있답니다.

영문자, 숫자, 언더스코어(_), 달러 기호($)만 사용 가능
숫자로 시작할 수 없음
예약어(JavaScript가 이미 사용하는 단어) 사용 불가
대소문자를 구분함

----------------------------------------------------------------------------------
// 올바른 변수명
let userName = "김개발";
let user_age = 20;
let $price = 1000;
let studentCount2 = 30;
----------------------------------------------------------------------------------

[ 데이터 타입, JavaScript가 다루는 데이터의 종류들 ]
----------------------------------------------------------------------------------
JavaScript에서 데이터 타입이란 변수에 저장할 수 있는 데이터의 종류를 말합니다.
마치 물건을 정리할 때 옷장, 신발장, 책장을 구분하듯이, 컴퓨터도 데이터의 성격에 따라 다르게 처리합니다.

JavaScript의 원시 타입(Primitive Types)은 가장 기본적인 데이터 타입들로, 총 5가지가 있습니다.

1. Number (숫자)
정수와 실수를 모두 포함하는 숫자 데이터입니다. 계산기처럼 수학적 연산이 가능합니다.

2. String (문자열)
텍스트 데이터로, 따옴표('')나 쌍따옴표("")로 감싸서 표현합니다. 책의 내용처럼 읽을 수 있는 문자들의 모음입니다.

3. Boolean (불린)
참(true) 또는 거짓(false)만을 나타내는 논리 데이터입니다. 전등의 켜짐/꺼짐 스위치와 같습니다.

4. Undefined (정의되지 않음)
변수가 선언되었지만 아직 값이 할당되지 않은 상태입니다. 빈 상자와 같습니다.

5. Null (널)
의도적으로 "값이 없음"을 나타내는 데이터입니다. 빈 상자라고 명시적으로 표시해둔 것과 같습니다.

----------------------------------------------------------------------------------
연산자, 데이터를 가공하는 도구들
----------------------------------------------------------------------------------
연산자는 데이터를 가공하고 조작하는 도구입니다.
마치 요리할 때 칼로 자르고, 팬으로 볶고, 믹서로 갈듯이,
프로그래밍에서도 다양한 연산자로 데이터를 원하는 형태로 변환합니다.

[ JavaScript의 주요 연산자들 ]
1. 산술 연산자 (Arithmetic Operators)
수학 계산을 위한 기본 도구들입니다. 계산기의 버튼들과 똑같다고 생각하면 됩니다.

2. 할당 연산자 (Assignment Operators)
변수에 값을 저장하거나 기존 값을 변경할 때 사용합니다. 마치 상자에 물건을 넣는 것과 같습니다.

3. 비교 연산자 (Comparison Operators)
두 값을 비교해서 참(true) 또는 거짓(false)을 판단합니다. 저울로 무게를 비교하는 것과 비슷합니다.

이 연산자들을 잘 활용하면 사용자 입력을 처리하고, 조건을 판단하고, 복잡한 계산을 수행할 수 있습니다.
웹 애플리케이션의 핵심 로직을 만드는 필수 도구들이죠!

----------------------------------------------------------------------------------
문자열 다루기, 텍스트 조작의 마법사
----------------------------------------------------------------------------------
문자열(String)은 텍스트 데이터를 다루는 JavaScript의 핵심 데이터 타입입니다.
마치 진주 목걸이처럼 여러 글자가 순서대로 연결되어 있는 형태라고 생각하면 됩니다.

[ 문자열 생성 방법 ]
JavaScript에서 문자열을 만드는 방법은 세 가지가 있습니다.

1. 작은따옴표(''): 가장 기본적인 방법
2. 큰따옴표(""): 작은따옴표와 동일한 기능
3. 백틱(``): ES6에서 추가된 템플릿 리터럴

[ 세 가지 문자열 생성 방법 ]
let message1 = '안녕하세요!';
let message2 = "반갑습니다!";
let message3 = `환영합니다!`;

[ 템플릿 리터럴 ]
템플릿 리터럴은 백틱(`)을 사용하는 특별한 문자열입니다.
마치 편지 양식처럼 미리 틀을 만들어두고 필요한 부분만 바꿔 넣을 수 있어요.
----------------------------------------------------------------------------------
let userName = "김개발";
let productName = "MacBook Pro";

// 기존 방식 (번거로움)
let oldMessage = userName + "님이 선택하신 " + productName;

// 템플릿 리터럴 (간편함)
let newMessage = `${userName}님이 선택하신 ${productName}`;
----------------------------------------------------------------------------------
${}안에 변수나 표현식을 넣으면 자동으로 그 값이 문자열에 삽입됩니다.
마치 빈칸 채우기 문제의 정답을 넣는 것처럼 말이죠!

< 내용 요약 >
1. 문자열 생성: 작은따옴표, 큰따옴표, 백틱 세 가지 방법
2. 템플릿 리터럴: 백틱과 ${}를 사용한 동적 문자열 생성
3. 문자열 연결: + 연산자보다 템플릿 리터럴이 더 편리
4. 실무 활용: 사용자 개인화 메시지, 동적 콘텐츠 생성에 필수

----------------------------------------------------------------------------------
조건문, 프로그램의 두뇌 역할하기
----------------------------------------------------------------------------------
조건문은 프로그램의 두뇌 역할을 합니다.
마치 우리가 일상생활에서 "만약 ~라면 ~하고, 그렇지 않으면 ~한다"고 판단하는 것처럼,
프로그램도 특정 조건에 따라 다른 동작을 수행할 수 있게 해주는 문법입니다.

JavaScript에서 가장 기본적인 조건문은 if문입니다.
if문의 구조를 집 현관문에 비유해보겠습니다.

현관문 앞에서 → 조건 확인 (열쇠가 있나?)
 ↓
있다면 → 문을 열고 들어간다 (if 블록 실행)
 ↓
없다면 → 벨을 누른다 (else 블록 실행)
----------------------------------------------------------------------------------
[ if문의 기본 구조 ]
----------------------------------------------------------------------------------
if (조건식) {
 // 조건이 참(true)일 때 실행되는 코드
} else {
 // 조건이 거짓(false)일 때 실행되는 코드
}
조건식은 반드시 true 또는 false로 판별되어야 합니다.
이를 불린(Boolean) 값이라고 하며, 프로그램이 "예/아니오"를 판단하는 기준이 됩니다.
----------------------------------------------------------------------------------
[ 비교 연산자들 ]
----------------------------------------------------------------------------------
조건을 만들 때 사용하는 주요 비교 연산자들입니다.

=== : 완전히 같다 (값과 타입 모두 같음)            ( == 가 아님! )
!== : 완전히 다르다
> : 크다
< : 작다
>= : 크거나 같다
<= : 작거나 같다
----------------------------------------------------------------------------------
[ 내용정리 ]
- if (조건식) { } 기본 구조
- else if로 여러 조건 처리
- === 연산자로 정확한 비교
- 조건식은 항상 true/false로 판별

----------------------------------------------------------------------------------
반복문, 같은 일을 효율적으로 처리하기
----------------------------------------------------------------------------------
반복문은 같은 작업을 여러 번 반복할 때 사용하는 JavaScript의 핵심 문법입니다.
마치 공장의 컨베이어 벨트처럼, 정해진 규칙에 따라 동일한 작업을 계속 수행합니다.
가장 기본적인 반복문은 for문입니다. for문의 구조를 살펴보겠습니다.
----------------------------------------------------------------------------------
for (초기값; 조건; 증감) {
    // 반복할 코드
}
----------------------------------------------------------------------------------
이를 실생활에 비유하면 다음과 같습니다.
- 초기값: "1번부터 시작해"
- 조건: "10번까지 계속해"
- 증감: "한 번 할 때마다 1씩 늘려"
- 반복할 코드: "실제로 해야 할 일"

예를 들어, 1부터 5까지 숫자를 출력하는 코드는 이렇게 작성합니다.
----------------------------------------------------------------------------------
for (let i = 1; i <= 5; i++) {
    console.log(i + "번째 반복");
}
----------------------------------------------------------------------------------
이 코드는 다음과 같이 실행됩니다.
1. i = 1로 시작
2. i <= 5 조건 확인 (참이므로 계속)
3. 코드 실행: "1번째 반복" 출력
4. i++로 i를 1 증가 (i = 2)
5. 조건 확인하고 반복...
----------------------------------------------------------------------------------
핵심 내용 요약:
- for문은 같은 작업을 여러 번 반복할 때 사용
- for (초기값; 조건; 증감) 구조로 작성
- 조건이 거짓이 되면 반복 종료
- 웹 페이지에서 동적 콘텐츠 생성에 매우 유용

배열, 여러 데이터를 한 번에 관리하기
----------------------------------------------------------------------------------
배열(Array)은 여러 개의 값을 순서대로 저장할 수 있는 데이터 구조입니다.
마치 아파트의 호수처럼..
하나의 건물(배열 이름) 안에 여러 개의 방(요소)이 있고,
각 방에는 고유한 번호(인덱스)가 있다고 생각하면 됩니다.

[ 배열의 특징 ]
- 순서가 있는 데이터 집합: 데이터가 입력된 순서대로 저장
- 인덱스로 접근: 0부터 시작하는 번호로 각 요소에 접근
- 동적 크기: 필요에 따라 크기를 늘리거나 줄일 수 있음
- 다양한 타입 저장: 숫자, 문자열, 불린 등 다양한 데이터 타입을 함께 저장 가능

[ 배열 생성 방법 ]
방법 1: 대괄호를 사용한 리터럴 방식 (가장 많이 사용)
let scores = [85, 90, 78, 92, 88];
let fruits = ['사과', '바나나', '오렌지'];

방법 2: Array 생성자 사용
let numbers = new Array(1, 2, 3, 4, 5);
let emptyArray = new Array(5); // 크기가 5인 빈 배열

방법 3: 빈 배열 생성 후 값 추가
let students = [];
students[0] = '김철수';
students[1] = '이영희';

[ 내용 정리 ]
배열은 [] 대괄호로 생성하고, 인덱스는 0부터 시작
array.length로 배열의 크기 확인
반복문과 함께 사용하면 모든 요소를 효율적으로 처리 가능
push(), pop() 등의 메서드로 동적 관리 가능

----------------------------------------------------------------------------------
객체(Object), 관련된 데이터를 묶어서 관리하기
----------------------------------------------------------------------------------
객체(Object)는 관련된 데이터와 기능을 하나로 묶어서 관리하는 JavaScript의 핵심 개념입니다.
실생활로 비유하면..
사람이라는 객체는 이름, 나이, 키, 몸무게 같은 속성(Property)과
걷기, 말하기, 먹기 같은 메서드(Method)를 가지고 있죠.
avaScript에서 객체를 만드는 방법은 여러 가지가 있지만,
가장 기본적인 방법은 객체 리터럴(Object Literal) 방식입니다.

[ 중괄호 {}를 사용해서 객체 생성 ]
----------------------------------------------------------------------------------
let student = {
 name: "김개발",                          ( 속성명 : 값 )
 age: 20,
 major: "정보통신공학과",
 Active: true
};
----------------------------------------------------------------------------------
객체의 속성에 접근하는 방법은 두 가지가 있습니다.

점 표기법(Dot Notation): 객체.속성명                 ( 예시: student.name )
대괄호 표기법(Bracket Notation): 객체["속성명"]       ( 예시 : student[name] )

점 표기법이 더 간단하고 읽기 쉬워서 주로 사용하지만,
속성명에 공백이 있거나 변수로 접근해야 할 때는 대괄호 표기법을 써야합니다.

[ 내용 정리 ]
- 객체는 {} 중괄호로 생성합니다
- 속성 접근: 객체.속성명 또는 객체["속성명"]
- 객체는 속성과 메서드를 모두 가질 수 있습니다
- this 키워드로 객체 자신을 참조합니다

----------------------------------------------------------------------------------
스코프와 실행 컨텍스트, 변수의 생명주기 이해하기
----------------------------------------------------------------------------------
스코프는 변수나 함수가 접근 가능한 범위를 의미합니다. 마치 아파트 단지처럼 생각해보세요.
각 세대(함수)는 자신만의 공간이 있고, 공용 공간(전역)도 있죠.
----------------------------------------------------------------------------------
전역 스코프 (Global Scope)
----------------------------------------------------------------------------------
아파트 단지의 공용 공간과 같습니다. 모든 곳에서 접근할 수 있어요.
----------------------------------------------------------------------------------
// 전역 스코프에 선언된 변수
let apartmentName = "해피빌리지";
var totalFloors = 15;

function showApartmentInfo() {

----------------------------------------------------------------------------------
에러 처리와 디버깅, 문제 해결의 달인 되기
----------------------------------------------------------------------------------
에러는 개발자의 영원한 동반자입니다.
마치 요리할 때 가끔 음식이 타거나 짜지는 것처럼...
코드를 작성하다 보면 반드시 에러를 만나게 되죠.
하지만 걱정하지 마세요!
에러는 우리의 적이 아니라 "뭔가 잘못됐다"고 알려주는 친절한 안내자입니다.

JavaScript에서 만날 수 있는 주요 에러들을 살펴보겠습니다.

1. SyntaxError (문법 에러)
코드의 문법이 잘못되었을 때 발생합니다. 마치 한국어 문장에서 띄어쓰기나 맞춤법이 틀린 것과 같아요.

2. ReferenceError (참조 에러)
존재하지 않는 변수나 함수를 사용하려 할 때 발생합니다. 전화번호부에 없는 사람에게 전화를 걸려는 것과 비슷하죠.

3. TypeError (타입 에러)
잘못된 타입의 값에 대해 연산을 수행하려 할 때 발생합니다. 숫자가 아닌데 수학 계산을 하려는 것처럼요.

4. RangeError (범위 에러)
숫자가 허용된 범위를 벗어났을 때 발생합니다.

에러를 다루는 가장 좋은 방법은 try-catch문을 사용하는 것입니다.
이는 마치 "혹시 문제가 생기면 이렇게 처리해줘"라고 미리 준비해두는 안전장치와 같아요.
