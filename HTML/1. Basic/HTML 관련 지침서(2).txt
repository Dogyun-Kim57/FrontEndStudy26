-----------------------------------------------------------------------------
폼(Form)이란?
-----------------------------------------------------------------------------
폼은 웹페이지에서 사용자가 정보를 입력하고 서버로 전송할 수 있게 해주는 HTML의 핵심 기능입니다.
마치 종이 설문지와 같은 역할을 한다고 생각하면 쉬워요!

"종이 설문지를 생각해보세요. "설문지에는 이름을 쓸 칸, 체크박스, 선택지들이 있죠?
그리고 마지막에 '제출' 버튼이 있어서 작성한 내용을 담당자에게 전달하게 됩니다."

웹의 폼도 정확히 같은 원리입니다.
- 입력 칸들: 사용자가 정보를 입력하는 공간
- 제출 버튼: 입력한 정보를 서버로 보내는 버튼
- 폼 태그: 이 모든 것을 하나로 묶어주는 컨테이너

form 태그의 기본 구조
-----------------------------------------------------------------------------
<form action="전송할_주소" method="전송_방법">
 <!-- 여기에 입력 요소들이 들어갑니다 -->
 <input type="text" name="username">
 <button type="submit">제출</button>
</form>
-----------------------------------------------------------------------------
핵심 속성 설명:
폼(Form)은 사용자 입력을 받아 서버로 전송하는 HTML 요소입니다
<form> 태그로 입력 요소들을 감싸고, action과 method 속성을 설정합니다
<input> 태그로 다양한 형태의 입력 칸을 만들 수 있습니다
<button type="submit">으로 폼 데이터를 전송합니다

- action: 폼 데이터를 어디로 보낼지 정하는 주소 (목적지)
- method: 데이터를 어떤 방식으로 보낼지 정하는 방법 (GET 또는 POST)

-----------------------------------------------------------------------------
텍스트 입력창 만들기
-----------------------------------------------------------------------------
텍스트 입력창이란 마치 종이 위의 빈 칸과 같습니다.
설문지를 작성할 때 "이름: ______" 이런 빈 칸에 펜으로 글씨를 쓰는 것처럼,
웹에서는 <input type="text">가 그 역할을 합니다.

input 태그의 특징:
- 단일 태그: <input>으로 시작해서 바로 끝남 (닫는 태그 없음)
- type 속성: 어떤 종류의 입력을 받을지 결정
- name 속성: 서버로 데이터를 보낼 때 사용하는 이름표
- placeholder: 사용자에게 힌트를 주는 안내 문구
- maxlength: 최대 입력 글자 수 제한
- required: 필수 입력 항목 지정
- size: 입력창의 크기 조절
- value: 기본값 설정
- readonly: 읽기 전용 (수정 불가)

핵심 포인트:
- name 속성은 필수 (서버로 데이터 전송시 사용)
- placeholder로 사용자에게 친절한 안내 제공
- required, maxlength 등으로 입력 검증 강화
- <input type="password" id="password" name="password" = 비밀번호 등 민감정보 입력해야되는 텍스트에 활용
- type="hidden": 사용자에게는 보이지 않지만 서버로 전송되는 정보
- 보안을 위해 비밀번호 필드에는 minlength, required 속성 활용
- 숨겨진 필드는 추가 정보 전달에 유용

-----------------------------------------------------------------------------
체크박스 및 라디오 버튼 만들기
-----------------------------------------------------------------------------
체크박스 vs 라디오버튼의 차이점
웹에서 사용자의 선택을 받는 방법은 크게 두 가지입니다.

체크박스 (Checkbox)
- 여러 개 선택 가능 (다중 선택)
- 각각 독립적으로 체크/해제
- 취미, 관심사, 약관 동의 등에 활용
- input type="checkbox"로 구현

라디오버튼 (Radio Button)
- 하나만 선택 가능 (단일 선택)
- 하나를 선택하면 다른 것은 자동 해제
- 성별, 학년, 결제 방법 등에 활용
- input type="radio"로 구현

HTML 구조의 핵심
선택 요소들을 만들 때 가장 중요한 것은 그룹핑입니다.

체크박스: 각각 다른 name 속성 사용 (독립적)
라디오버튼: 같은 name 속성 사용 (그룹으로 묶임)
label 태그: 클릭 영역 확대와 접근성 향상

핵심 정리
- 체크박스: 여러 선택 가능, 각각 다른 name 속성
- 라디오버튼: 하나만 선택, 같은 name 속성으로 그룹핑
- label 태그: 접근성과 사용성 향상을 위해 필수 사용
- required 속성: 필수 선택 항목 지정 가능

-----------------------------------------------------------------------------
드롭다운 기능 및 다중 선택 기능
-----------------------------------------------------------------------------
드롭다운(dropdown)은 마치 접이식 서랍과 같습니다. 평소에는 하나의 선택된 값만 보이다가,
클릭하면 여러 선택지가 펼쳐지는 방식이죠.
HTML에서는 <select>와 <option> 태그의 조합으로 이를 구현할 수 있습니다.

<select> 태그는 드롭다운 상자 전체를 의미하고,
<option> 태그는 각각의 선택 항목을 나타냅니다. 이는 마치 메뉴판과 메뉴 항목의 관계와 같습니다.

< 예시 >
-----------------------------------------------------------------------------
<select name="region">
 <option value="seoul">서울특별시</option>
 <option value="busan">부산광역시</option>
 <option value="incheon">인천광역시</option>
</select>
-----------------------------------------------------------------------------
여기서 중요한 점은 value 속성입니다.
사용자에게는 "서울특별시"라고 보이지만,
실제로 서버에 전송되는 값은 "seoul"입니다. 이는 데이터베이스에서 처리하기 편한 영문 코드를 사용하기 위함입니다.

또한 <select> 태그에는 여러 유용한 속성들이 있습니다.
- multiple: 여러 항목을 동시에 선택 가능
- size: 한 번에 보여줄 항목의 개수 지정
- disabled: 선택 불가능하게 만들기

핵심정리
<select>와 <option>: 드롭다운의 기본 구조
value 속성: 실제 전송되는 데이터 값
<optgroup>: 관련 항목들의 그룹화
multiple 속성: 다중 선택 기능
selected 속성: 기본 선택값 지정

긴글 등 입력하는 폼 만들기 ( 요청사항 기재 등을 만들때 활용 )
-----------------------------------------------------------------------------
해당 폼을 만들때 활용하는 textarea는 여러 줄 텍스트의 전용 공간
textarea 태그는 마치 디지털 원고지와 같습니다.
일반 input 태그가 한 줄짜리 메모지라면, textarea는 여러 페이지에 걸쳐 긴 글을 쓸 수 있는 노트북이죠.

-----------------------------------------------------------------------------
[예시1]
<textarea></textarea>
-----------------------------------------------------------------------------
이렇게 간단하게 시작할 수 있지만, 실제로는 더 많은 속성들이 필요합니다.

-----------------------------------------------------------------------------
[예시2]
<textarea rows="5" cols="50" placeholder="문의 내용을 입력해주세요" maxlength="500"></textarea>
-----------------------------------------------------------------------------
rows: 기본적으로 보여줄 줄 수 (세로 크기)
cols: 기본적으로 보여줄 글자 수 (가로 크기)
placeholder: 입력 전 보여줄 안내 문구
maxlength: 최대 입력 가능한 글자 수

input 태그와의 차이점

일반 input 태그는 자기 완결적(self-closing)이지만, textarea는 여는 태그와 닫는 태그가 분리되어 있습니다.
그 이유는 기본값을 태그 사이에 넣을 수 있기 때문입니다.

-----------------------------------------------------------------------------
[예시3]
<textarea rows="3" cols="40">
여기에 미리 입력된 내용이 나타납니다.
사용자가 수정할 수 있어요.
</textarea>
-----------------------------------------------------------------------------

핵심정리
textarea는 긴 글 입력을 위한 전용 태그
rows, cols 속성으로 기본 크기 설정
placeholder와 maxlength로 사용자 경험 향상
CSS의 resize 속성으로 크기 조절 방식 제어
