-------------------------------------------------------------
HTML5 시맨틱 태그
-------------------------------------------------------------
div만으로 구성된 웹페이지와 달리...
시맨틱 태그를 사용하면 문서의 구조가 명확해지고 검색 엔진과 스크린 리더가 콘텐츠를 더 잘 이해할 수 있습니다.
헤더, 네비게이션, 메인 콘텐츠, 푸터 등 웹페이지의 각 영역에 적절한 의미를 부여하는 방법을 배워봅니다.

◆해당 용어에 대해 짧게 알고 가기◆
01. 웹페이지에 의미를 부여하는 시맨틱의 힘
시맨틱 태그의 개념과 div 기반 레이아웃 대비 장점을 알아봅니다.

02. 페이지의 머리, header로 시작하기
header 태그로 페이지나 섹션의 머리말 영역을 정의합니다.

03. 길잡이 역할의 nav, 사용자를 안내하다
nav 태그로 내비게이션 링크 영역을 구성합니다.

04. 페이지의 핵심, main으로 집중하기
main 태그로 페이지의 주요 콘텐츠 영역을 명확히 표시합니다.

05. 내용을 나누는 section, 주제별로 정리하기
section 태그로 관련 콘텐츠를 주제별로 그룹화합니다.

06. 독립적인 콘텐츠, article로 완성하기
article 태그로 독립적으로 배포 가능한 콘텐츠를 정의합니다.

07. 보조 정보의 공간, aside로 채우기
aside 태그로 사이드바, 광고, 관련 링크 등 부가 정보 영역을 만듭니다.

08. 페이지의 마무리, footer로 완성하기
footer 태그로 저작권 정보, 연락처 등 페이지 하단 영역을 구성합니다.

09. 시맨틱 웹의 완성, 모든 것을 하나로
학습한 시맨틱 태그들을 조합하여 완전한 웹페이지 구조를 설계합니다.
-------------------------------------------------------------

-------------------------------------------------------------
시맨틱(Semantic)이란?
-------------------------------------------------------------
시맨틱은 '의미론적'이라는 뜻입니다.
웹 개발에서 시맨틱 마크업이란 HTML 태그가 단순히 모양을 만드는 것이 아니라,
콘텐츠의 의미를 명확히 표현하는 것을 말합니다.

"밥그릇, 국그릇, 반찬그릇이 모두 다른 모양인 이유가 뭘까? 각각의 용도와 의미가 다르기 때문이지. HTML도 마찬가지!"
-------------------------------------------------------------
＊시맨틱 마크업의 장점＊
-------------------------------------------------------------
1. 접근성 향상: 스크린 리더 같은 보조 기술이 콘텐츠를 정확히 해석
2. SEO 최적화: 검색엔진이 페이지 구조와 내용을 더 잘 이해
3. 코드 가독성: 개발자가 코드의 의미를 쉽게 파악
4. 유지보수성: 구조가 명확해 수정과 관리가 용이
-------------------------------------------------------------


-------------------------------------------------------------
header 태그란 무엇인가?
-------------------------------------------------------------
<header> 태그는 HTML5에서 도입된 시맨틱 태그 중 하나로, 문서나 섹션의 머리말 영역을 의미합니다.
마치 신문의 헤드라인처럼, 웹페이지에서 가장 중요한 정보가 담기는 영역이죠.

책을 펼쳤을 때 제목과 저자명이 맨 위에 있듯이,
웹사이트도 로고, 회사명, 주요 메뉴가 맨 위에 있어야 사용자가 '아, 내가 지금 어디에 있구나'를 바로 알 수 있듯이..

< header 태그의 특징 >
의미적 명확성: div 태그와 달리 '머리말'이라는 명확한 의미를 가집니다.
접근성 향상: 스크린 리더 같은 보조 기술이 페이지 구조를 더 잘 이해합니다.
SEO 최적화: 검색 엔진이 페이지의 중요한 정보를 더 쉽게 찾을 수 있습니다.
유연한 사용: 전체 페이지뿐만 아니라 개별 섹션의 헤더로도 사용 가능합니다.

< header에 들어가는 일반적인 내용 >
1. 로고 또는 사이트명
2. 주요 네비게이션 메뉴
3. 검색 기능
4. 연락처 정보
5. 소셜 미디어 링크
-------------------------------------------------------------


-------------------------------------------------------------
03. 길잡이 역할의 nav, 사용자를 안내하다
-------------------------------------------------------------
nav 태그는 Navigation의 줄임말로, 웹페이지에서 주요 내비게이션 링크들을 그룹화하는 시맨틱 태그입니다.
마치 건물의 안내판이나 지하철 노선도처럼, 사용자가 원하는 곳으로 쉽게 이동할 수 있도록 도와주는 역할을 합니다.

< nav 태그가 특별한 이유 >
일반적인 <div> 태그로도 메뉴를 만들 수 있지만, nav 태그를 사용하면 다음과 같은 장점이 있습니다.

1. 의미 전달: 검색엔진과 스크린 리더가 "여기가 내비게이션 영역이구나!"라고 명확히 인식
2. 접근성 향상: 시각 장애인을 위한 스크린 리더가 내비게이션을 쉽게 찾을 수 있음
3. SEO 최적화: 검색엔진이 사이트 구조를 더 잘 이해하여 검색 순위에 도움
4. 유지보수성: 개발자가 코드를 볼 때 내비게이션 영역을 한눈에 파악 가능


다만, nav 태그는 모든 링크에 사용하는 게 아님. '주요한 내비게이션'에만 사용해야 함.
예를 들어, 메인 메뉴, 브레드크럼(경로 표시), 페이지네이션 등..
footer의 작은 링크들까지 nav로 감쌀 필요는 없음!
-------------------------------------------------------------


-------------------------------------------------------------
페이지의 핵심, main으로 집중하기
-------------------------------------------------------------
main 태그는 웹페이지의 주요 내용을 담는 컨테이너입니다.
마치 신문에서 1면 기사처럼, 페이지에서 가장 중요하고 핵심적인 내용을 포함하는 영역이죠.


< main 태그의 특별한 점들 >

1. 페이지당 하나만 사용 가능함

< 예시 >
-------------------------------------------------------------
<main>
 <h1>오늘의 뉴스</h1>
 <article>...</article>
</main>
-------------------------------------------------------------
< 잘못된 예시.. main은 하나만! >
<main>첫 번째 메인</main>
<main>두 번째 메인</main>
-------------------------------------------------------------
이는 마치 연극 무대에서 주연 배우가 한 명인 것과 같은 원리입니다.
여러 명이 동시에 주연을 할 수는 없으니까요.

2. 반복되지 않는 고유한 내용
main 태그 안에는 해당 페이지에서만 볼 수 있는 고유한 내용이 들어가야 합니다.
모든 페이지에 공통으로 나타나는 header, footer, aside 등은 main 밖에 위치해야 함.

3. 접근성 향상의 핵심
스크린 리더 같은 보조 기술이 "이 페이지의 핵심 내용은 여기에 있어요!"라고 바로 찾아갈 수 있게 도와줍니다.
-------------------------------------------------------------


-------------------------------------------------------------
내용을 나누는 section, 주제별로 정리하기
-------------------------------------------------------------
<section> 태그는 문서 내에서 주제별로 관련된 내용을 그룹화하는 시맨틱 태그입니다.
마치 책의 장(章)과 같은 역할을 한다고 생각하면 쉽습니다.

section 태그의 특징
1. 주제별 구분
- 하나의 독립적인 주제나 내용을 담습니다
- 각 section은 고유한 제목을 가져야 합니다
- 일반적으로 <h1> ~ <h6> 태그로 시작합니다


2. div와의 차이점
< 예시 >
-------------------------------------------------------------
[ div: 단순한 그룹화 (의미 없음) ]
<div class="company-info">
 <h2>회사 소개</h2>
 <p>저희 회사는...</p>
</div>
-------------------------------------------------------------
[ section: 의미 있는 주제별 구분 ]
<section>
 <h2>회사 소개</h2>
 <p>저희 회사는...</p>
</section>
-------------------------------------------------------------
3. 사용 기준
- 독립적인 주제가 있을 때
- 제목이 필요할 때
- 목차에 들어갈 만한 내용일 때
- 단순한 스타일링 목적이라면 <div> 사용

section은 '이 부분은 무엇에 대한 내용인가?'라고 물었을 때 명확하게 답할 수 있는 내용을 담아야 합니다.
예를 들어 '회사 소개', '제품 설명', '연락처' 이런 식으로 말이죠.
-------------------------------------------------------------


-------------------------------------------------------------
독립적인 콘텐츠, article로 완성하기
-------------------------------------------------------------
article 태그란?
article 태그는 독립적으로 배포하거나 재사용할 수 있는 완전한 콘텐츠를 나타내는 시맨틱 태그입니다.
마치 신문의 기사처럼..
그 자체로 완전한 의미를 가지고 다른 곳으로 옮겨도 이해할 수 있는 콘텐츠에 사용합니다.

< section vs article, 어떻게 구분할까? >

section: "책의 챕터"처럼 전체 맥락에서 의미를 가지는 구역
article: "잡지의 기사"처럼 그 자체로 완전한 독립적인 콘텐츠

-------------------------------------------------------------
< 예시 >
-------------------------------------------------------------
[ section 예시: 전체 맥락이 필요함 ]

<section>
 <h2>1장. HTML 기초</h2>
 <p>HTML은 웹페이지의 구조를 만드는...</p>
</section>

-------------------------------------------------------------
[ article 예시: 독립적으로 의미가 완전함 ]

<article>
 <h2>웹 개발자가 되는 5가지 방법</h2>
 <p>웹 개발자가 되고 싶다면...</p>
 <footer>작성자: 김개발</footer>
</article>

-------------------------------------------------------------
< article을 사용해야 하는 경우 >
블로그 포스트: RSS 피드로 배포 가능
뉴스 기사: 다른 사이트에서 인용 가능
제품 리뷰: 쇼핑몰에서 독립적으로 표시
포럼 게시글: 검색 결과에서 단독으로 노출
댓글: 해당 댓글만으로도 의미 파악 가능
"핵심은 '이 콘텐츠를 다른 곳에 복사해서 붙여넣어도 독자가 이해할 수 있는가?'입니다
-------------------------------------------------------------


-------------------------------------------------------------
보조 정보의 공간, aside로 채우기
-------------------------------------------------------------
aside 태그는 HTML5에서 보조적인 콘텐츠를 나타내는 시맨틱 태그입니다.
마치 신문의 사이드바나 잡지의 박스 기사처럼,
메인 콘텐츠와 직접적인 관련은 있지만 독립적으로 존재할 수 있는 정보를 담습니다.

aside 태그의 특징
1. 보조적 역할
- 메인 콘텐츠를 보완하는 정보
- 없어도 메인 콘텐츠 이해에 지장 없음
- 하지만 있으면 더 풍부한 경험 제공

2. 다양한 활용 용도
-------------------------------------------------------------
< 예시 >
[ 블로그 사이드바 ]
<aside class="sidebar">
 <h3>인기 글</h3>
 <h3>태그 모음</h3>
 <h3>광고</h3>
</aside>
-------------------------------------------------------------
[ 본문 중 관련 정보 ]
<article>
 <p>웹 개발을 배우면서...</p>
 <aside class="tip">
 <h4> 개발 팁</h4>
 <p>HTML 태그는 의미를 생각하며 사용하세요!</p>
 </aside>
</article>
-------------------------------------------------------------
3. 위치의 자유로움
aside는 페이지의 사이드바일 수도 있고, 본문 중간의 박스 형태일 수도 있습니다.
마치 보조 배우처럼 어디든 필요한 곳에 등장할 수 있음!


< 언제 aside를 사용할까? >
블로그/뉴스 사이트의 사이드바
관련 글 추천
광고 영역
용어 설명 박스
인용구나 참고 자료
작성자 정보
-------------------------------------------------------------


-------------------------------------------------------------
페이지의 마무리, footer로 완성하기
-------------------------------------------------------------
<footer> 태그는 페이지나 섹션의 바닥글(footer) 정보를 담는 시맨틱 태그입니다.
마치 편지의 서명란이나 책의 마지막 페이지처럼, 웹페이지에서 마무리 정보를 제공하는 역할을 합니다.

footer의 주요 특징
1. 위치의 유연성
- 페이지 전체의 하단에 올 수도 있고
- 특정 섹션의 하단에 올 수도 있습니다
- 여러 개의 footer가 한 페이지에 존재할 수 있어요

2. 일반적인 내용들
< 예시 >
-------------------------------------------------------------
[ 전형적인 footer 내용들 ]
<footer>
 <p>&copy; 2024 우리회사. 모든 권리 보유.</p>
 <nav>
 <a href="/about">회사소개</a>
 <a href="/contact">연락처</a>
 <a href="/privacy">개인정보처리방침</a>
 </nav>
 <address>
 서울시 강남구 테헤란로 123
 전화: 02-1234-5678
 </address>
</footer>
-------------------------------------------------------------
3. 시맨틱의 중요성
단순히 <div class="footer">를 사용하는 것보다 <footer> 태그를 사용하면
- 검색엔진이 페이지 구조를 더 잘 이해합니다
- 스크린 리더가 시각장애인에게 더 정확한 정보를 제공합니다
- 코드 가독성이 훨씬 좋아집니다

footer는 마치 명함을 예시로 들 수 있습니다.
작지만 중요한 정보들이 깔끔하게 정리되어 있어야 합닏다.
사용자가 마지막에 보는 부분이기에 인상을 더 강하게 남겨야 되니까요.
-------------------------------------------------------------


-------------------------------------------------------------
시맨틱 웹의 완성, 모든 것을 하나로
-------------------------------------------------------------
시맨틱 웹의 완성은 모든 시맨틱 태그가 조화롭게 구성된 하나의 완전한 문서 구조를 만드는 것입니다.
이는 마치 잘 설계된 건물과 같습니다.

그렇기에 시맨틱 태그의 진짜 힘은 각각을 따로 쓸 때가 아니라
모든 태그가 하나로 어우러져 완전한 의미 구조를 만들 때 발휘됩니다.
마치 오케스트라처럼 말이죠.

< 시맨틱 태그의 계층 구조 >

 전체 페이지 (body)
├── header (사이트 제목, 로고)
├── nav (주요 메뉴)
├── main (주요 콘텐츠 영역)
│ ├── section (콘텐츠 섹션들)
│ │ └── article (개별 콘텐츠)
│ └── aside (부가 정보)
└── footer (저작권, 연락처)

1.header: 사이트 전체의 제목과 소개
2.nav: 주요 메뉴로 사용자 안내
3.main: 핵심 콘텐츠 영역 (section + aside)
4.section: 블로그 포스트들의 모음
5.article: 개별 블로그 포스트
6.aside: 부가 정보 (인기 글, 태그)
7.footer: 사이트 정보와 연락처

이 구조는 검색엔진이 페이지의 내용을 정확히 이해할 수 있게 도와주며,
스크린 리더 사용자들이 페이지를 쉽게 탐색할 수 있게 합니다.
또한 CSS 스타일링과 JavaScript 작업도 훨씬 명확하고 유지보수하기 쉬워집니다.

각 태그는 고유한 역할을 가지면서도 서로 의미적으로 연결되어 있습니다.
예를 들어, nav는 main으로 이어지는 길잡이 역할을, aside는 main을 보완하는 역할을 합니다.
