---------------------------------------------------------------------------------
< CSS 스타일링이란? >
CSS(Cascading Style Sheets)는 웹페이지의 시각적 표현을 담당하는 스타일 언어입니다.
HTML이 집의 뼈대라면...
CSS는 페인트를 칠하고, 가구를 배치하고, 조명을 설치하는 인테리어 디자이너 역할을 합니다.
같은 HTML이라도 CSS에 따라 전혀 다른 느낌의 웹사이트가 탄생할 수 있죠.

< CSS의 핵심 특징들 >
1. 분리의 원칙: HTML은 내용(구조)에, CSS는 표현(디자인)에 집중
2. 재사용성: 하나의 CSS 파일로 여러 HTML 페이지의 스타일 관리 가능
3. 유지보수성: 디자인 변경 시 CSS만 수정하면 전체 사이트 스타일 변경 가능
4. Cascading: 여러 스타일 규칙이 적용될 때의 우선순위 시스템

가장 중요한 건 CSS를 사용하면 웹페이지가 정말 살아있는 것처럼 보인다는 겁니다.
색상, 글꼴, 레이아웃, 애니메이션까지...
사용자가 '와, 이 사이트 정말 멋지다!'라고 생각하게 만드는 모든 요소가 CSS에서 나옵니다.

- CSS = 웹페이지의 스타일과 레이아웃을 정의
- HTML과 CSS의 분리로 유지보수성 향상
- 같은 HTML도 CSS에 따라 완전히 다른 모습으로 변신 가능

---------------------------------------------------------------------------------
CSS를 HTML에 연결하는 세 가지 방법
---------------------------------------------------------------------------------
CSS를 HTML에 연결하는 방법은
1. 인라인 스타일(Inline Style)
2. 내부 스타일시트(Internal Stylesheet)
3. 외부 스타일시트(External Stylesheet)
이렇게 세 가지입니다.

1. 인라인 스타일 (Inline Style)
HTML 요소에 직접 style 속성을 사용해 CSS를 적용하는 방법입니다.
마치 메모지를 물건에 직접 붙이는 것과 같아요.

장점: 즉시 적용되고 우선순위가 가장 높음
단점: 코드가 복잡해지고 재사용이 어려움


2. 내부 스타일시트 (Internal Stylesheet)
HTML 문서의 <head> 태그 안에 <style> 태그를 사용해 CSS를 작성하는 방법입니다.
한 권의 책 안에 목차를 넣는 것과 비슷해요.

장점: 해당 HTML 파일 전체에 적용 가능
단점: 다른 HTML 파일에서는 사용할 수 없음


3. 외부 스타일시트 (External Stylesheet)
별도의 CSS 파일을 만들고 HTML에서 <link> 태그로 연결하는 방법입니다.
공용 규칙집을 만들어 여러 곳에서 참조하는 것과 같아요.

장점: 여러 HTML 파일에서 재사용 가능, 관리가 편함
단점: 별도 파일이 필요함

인라인 스타일: 급하게 하나만 바꿀 때 (남용 금지!)
내부 스타일시트: 한 페이지만의 특별한 스타일이 필요할 때
외부 스타일시트: 일반적인 실무 상황 (가장 권장!)

만약 적용 시 우선순위는 인라인 > 내부 > 외부 순으로 적용됨.
---------------------------------------------------------------------------------


---------------------------------------------------------------------------------
CSS 문법의 기초, 규칙을 이해하자
---------------------------------------------------------------------------------
CSS는 "누구를 어떻게 꾸밀 것인가"를 명확하게 표현하는 언어입니다.
마치 패션 스타일리스트가 "빨간 드레스를 입은 모델의 머리를 금발로 염색하고,
빨간 립스틱을 발라주세요"라고 지시하는 것과 같습니다.

CSS의 기본 문법 구조는 다음과 같습니다.

---------------------------------------------------------------------------------
선택자 {
 속성: 값;
 속성: 값;
}
---------------------------------------------------------------------------------
이를 CSS 규칙(Rule) 또는 룰셋(Ruleset)이라고 부릅니다. 각 구성 요소를 자세히 살펴보겠습니다.

1. 선택자(Selector)
"누구를" 꾸밀 것인지 지정하는 부분입니다. HTML 요소를 선택하는 방법을 정의합니다.

2. 선언 블록(Declaration Block)
중괄호 {} 안에 있는 전체 영역으로, 실제 스타일 지시사항들이 들어갑니다.

3. 선언(Declaration)
속성: 값; 형태로 구성된 하나의 스타일 지시사항입니다.

4. 속성(Property)
"무엇을" 바꿀 것인지 지정합니다. (예: color, font-size, background)

5. 값(Value)
속성을 "어떻게" 바꿀 것인지 구체적으로 명시합니다. (예: red, 16px, center)

즉, CSS 규칙은 마치 회사의 복장 규정과 같아요.
모든 직원은(선택자) 정장을 입고(속성), 검은색으로(값) 착용한다'처럼 명확하게 규칙을 정하는 거죠!

(예시)
---------------------------------------------------------------------------------
h1 {               /* 선택자 및 선언 블록 설정 */
  속성 :  값
 color: #ff6b6b; /* 헥스 색상 코드        색상이라는 속성을 #ff6b6b의 색(값)으로 바꾸겠다. */
 font-size: 32px; /* 글자 크기           글자크기라는 속성을 32px 크기(값)로 바꾸겠다. */
 text-align: center; /* 텍스트 정렬       텍스트라는 속성을 가운데 정렬이라는 값으로 바꾸겠다. */
 background-color: #f8f9fa; /* 배경색    배경색이라는 속성을 #f8fpfa 라는 색(값)으로 바꾸겠다. */
 padding: 20px; /* 안쪽 여백             안쪽 여백이라는 속성을 20px라는 크기(값)로 바꾸겠다. */
 border: 2px solid #dee2e6; /* 테두리    테두리라는 속성을 solid #dee2e6이라는 색(값)으로 바꾸겠다. */
}          /* 블록 선언 마무리 */
---------------------------------------------------------------------------------
- 각 선언은 반드시 세미콜론(;)으로 끝나야 합니다
- 속성과 값 사이에는 콜론(:)을 사용합니다
- 중괄호 {} 안에 선언들을 작성합니다

CSS 규칙 구조: 선택자 { 속성: 값; }
선택자: 스타일을 적용할 HTML 요소 지정
선언 블록: 중괄호 {} 안의 스타일 지시사항들
선언: 속성: 값; 형태의 개별 스타일 규칙
문법 규칙: 콜론(:)으로 속성과 값 연결, 세미콜론(;)으로 선언 종료
---------------------------------------------------------------------------------


---------------------------------------------------------------------------------
기본 선택자로 원하는 요소 찾기
---------------------------------------------------------------------------------
CSS 선택자는 HTML 문서에서 스타일을 적용할 요소를 찾는 방법입니다.
마치 주소록에서 특정 사람을 찾는 것처럼, 각각 다른 방식으로 요소를 식별할 수 있어요.

4가지 기본 선택자의 특징
1. 전체 선택자 (*)
- 모든 요소를 선택하는 만능 도구
- 마치 "전교생 집합!"과 같은 개념

2. 타입 선택자 (태그명)
- 같은 종류의 모든 요소를 선택
- "모든 제목(h1) 집합!" 같은 느낌

3. 클래스 선택자 (.클래스명)
- 같은 클래스를 가진 요소들을 선택
- "축구부 학생들 집합!" 처럼 그룹별 선택

4. ID 선택자 (#아이디명)
- 특정 ID를 가진 하나의 요소만 선택
- "학번 20240001번 학생!" 처럼 개별 지정

이 선택자들은 구체성(specificity)이라는 우선순위를 가집니다.
ID 선택자가 가장 강력하고, 전체 선택자가 가장 약해요.
마치 대통령 명령이 시장 명령보다 우선하는 것과 같은 원리죠!
---------------------------------------------------------------------------------


---------------------------------------------------------------------------------
속성 선택자로 더 정교하게 선택하기
---------------------------------------------------------------------------------
속성 선택자는 HTML 요소가 가진 속성(attribute)을 기준으로 스타일을 적용하는 강력한 도구입니다.
마치 도서관에서 "빨간 표지의 책", "2023년에 출간된 책"처럼 특정 조건을 만족하는 책들을 찾는 것과 같아요.

속성 존재 선택자 [속성명]
가장 기본적인 형태로, 특정 속성을 가진 요소를 선택합니다.

< 예시 >
---------------------------------------------------------------------------------
[ required 속성을 가진 모든 요소 ]
[required] {
 border: 2px solid red;
}

[ disabled 속성을 가진 모든 요소 ]
[disabled] {
 background-color: #f0f0f0;
 cursor: not-allowed;
}
---------------------------------------------------------------------------------

더 나아가 속상 안의 값이 일치하는 선택자를 지정하는것도 가능하다.

< 예시 >
---------------------------------------------------------------------------------
[ type이 "email"인 input 요소 ]
[type="email"] {
 background-image: url('email-icon.png');
}

[ class가 정확히 "button"인 요소 ]
[class="button"] {
 padding: 10px 20px;
}
---------------------------------------------------------------------------------
이외에도 다양한 속성 선택자가 있어요.
- [속성^="값"]: 값으로 시작하는 속성
- [속성$="값"]: 값으로 끝나는 속성
- [속성*="값"]: 값을 포함하는 속성
---------------------------------------------------------------------------------


---------------------------------------------------------------------------------
관계 선택자로 요소 간의 연결고리 찾기
---------------------------------------------------------------------------------
HTML 요소들은 마치 가족 관계처럼 서로 연결되어 있습니다.
이런 관계를 이용해서 스타일을 적용하는 것이 바로 관계 선택자입니다.
예를들어 div가 아버지라면, 그 안의 p는 자식이고, p끼리는 형제 관계죠.
이런 관계를 CSS에서 활용할 수 있습니다.

주요 관계 선택자 유형
1. 자손 선택자 (Descendant Selector)
- 문법: 부모요소 자손요소
- 의미: 부모 요소 안에 있는 모든 자손 요소를 선택
- 비유: "김씨 집안의 모든 후손들"

2. 자식 선택자 (Child Selector)
- 문법: 부모요소 > 자식요소
- 의미: 부모 요소의 직접적인 자식 요소만 선택
- 비유: "김씨의 직계 자녀들만"

3. 인접 형제 선택자 (Adjacent Sibling)
- 문법: 요소1 + 요소2
- 의미: 요소1 바로 다음에 오는 요소2를 선택
- 비유: "첫째 바로 다음의 둘째"

4. 일반 형제 선택자 (General Sibling)
- 문법: 요소1 ~ 요소2
- 의미: 요소1 이후에 오는 모든 형제 요소2를 선택
- 비유: "첫째 이후의 모든 형제들"
